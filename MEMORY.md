# MEMORY.md

This is a shadow entry point for the repo memory layer.

The canonical source of truth lives in `memory/README.md`.

## What belongs here
- durable repo knowledge and operator context
- distilled facts, preferences, and conventions
- working notes that can later be promoted or deleted

## Buckets
- `memory/daily/` for dated short-term notes
- `memory/long-term/` for stable knowledge
- `memory/scratch/` for temporary working notes

If this file and `memory/README.md` ever drift, treat `memory/README.md` as authoritative.
